**Research task 1 — Workday API mechanics**

Workday hosts company job boards at URLs like:
`https://browserstack.wd3.myworkdayjobs.com/en-US/External`

The job listings API is undocumented but community-known. Research and answer:

1. What is the exact POST endpoint structure? (I've seen patterns like `/wday/cxs/{tenant}/{board}/jobs` — confirm or correct)
2. What request headers are required? (especially `Content-Type`, any `X-Workday-Client` or CSRF headers?)
3. What is the full request body schema? (fields like `limit`, `offset`, any required filters?)
4. What does the response JSON look like? Give a representative sample with fields: job title, URL, location, department, posted date
5. What is the "board slug"? In `https://browserstack.wd3.myworkdayjobs.com/en-US/External`, which part is the board slug vs the tenant? Map this clearly.
6. Does this API require authentication or cookies from a browser session?
7. How stable is this API in practice — has it broken for scrapers/bots in 2024–2025?
8. Is there any official Workday public API alternative?
