Task 1 — Reverse Engineering the Workday Public Job API

> TL;DR: The public careers sites use an undocumented JSON API that has remained remarkably stable for several years. It is different from Workday's official authenticated REST APIs and is intended to power public job boards rather than enterprise integrations. 




---

1. Endpoint structure

The endpoint pattern is:

POST https://{tenant}.wd{N}.myworkdayjobs.com/wday/cxs/{tenant}/{board}/jobs

Examples:

https://browserstack.wd3.myworkdayjobs.com/wday/cxs/browserstack/External/jobs

https://visa.wd5.myworkdayjobs.com/wday/cxs/visa/Visa/jobs

https://barclays.wd3.myworkdayjobs.com/wday/cxs/barclays/External_Career_Site_Barclays/jobs

https://fractal.wd1.myworkdayjobs.com/wday/cxs/fractal/Careers/jobs

2. Required headers

The API is surprisingly lightweight.

Minimum headers that work for nearly every public board:

Content-Type: application/json

Accept: application/json

A browser also sends:

Origin:
https://browserstack.wd3.myworkdayjobs.com

Referer:
https://browserstack.wd3.myworkdayjobs.com/en-US/External

Most clients additionally include a normal User-Agent. Community examples show that browser-like headers (Origin, Referer, Accept, Content-Type) are sufficient; no X-Workday-Client or CSRF token is required for public job searches.

4. Response structure

A typical response looks like:

{
  "total": 143,
  "jobPostings": [
    {
      "bulletFields": [
        "Bengaluru, India"
      ],
      "title": "Senior Software Engineer",
      "externalPath":
        "/job/Bengaluru/Senior-Software-Engineer_JR12345",

      "locationsText":
        "Bengaluru, India",

      "postedOn":
        "Posted 12 Days Ago",

      "businessTitle":
        "Engineering",

      "timeType":
        "Full time"
    }
  ]
}

Useful fields

Field	Meaning

title	Job title
externalPath	Relative URL
locationsText	Human-readable location
postedOn	Display posting age/date
bulletFields	Extra metadata (often location or department)
businessTitle	Team / department (when present)
timeType	Full-time / Part-time


Some boards include additional fields depending on tenant configuration. Public responses can vary slightly because Workday allows tenant-specific customization.

8. Official API?

This is important.

The public jobs endpoint is not the official Workday REST API.

Workday's supported APIs are authenticated enterprise APIs (REST, SOAP, Graph) intended for customers and integrations, not anonymous public job boards. Those require tenant credentials and appropriate permissions.

My recommendation for Job Hunter

I would implement a generic configuration like:

class WorkdayConfig:
    tenant: str
    board: str
    language: str = "en-US"

Then generate URLs dynamically:

base = (
    f"https://{tenant}.wd{cluster}.myworkdayjobs.com"
)

board_url = (
    f"{base}/{language}/{board}"
)

api = (
    f"{base}/wday/cxs/{tenant}/{board}/jobs"
)

That makes the adapter reusable across almost every Workday-backed company.
